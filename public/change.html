Things that might be nice to do here:

  • Add password protection to the form submission. Easiest way I can think of is a password field of some kind. Only a specific value allows you to make a change. The API route would read the secret value to match from an environment variable.
  • Make it more clear whether changes were successful or not. Maybe the response from the "/api/handle-form" route can redirect based on success?
  • Show the build status. Should be possible with Vercel, at least. Maybe even, once the build finishes, redirect to the page that was previously edited? Maybe there's a way to show an estimated build time remaining as well?
  • If build status is in any kind of progress or queued state, prevent edits programmatically. For example, disable form inputs, and in API route, check for Vercel build status before trying to make a commit.
